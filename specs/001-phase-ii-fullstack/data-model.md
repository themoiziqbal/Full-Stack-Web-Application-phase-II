## Entities

### User
Represents an individual who can own and manage todos.
- `id`: UUID (Primary Key) - Generated by Better Auth / DB
- `email`: String (Unique, Required) - User email for authentication
- `name`: String (Optional) - User's display name
- `created_at`: DateTime - Timestamp of account creation

### Todo
Represents a task item belonging to a specific user.
- `id`: Integer (Primary Key, Auto-increment)
- `user_id`: UUID (Foreign Key -> User.id) - Owner of the task
- `title`: String (Required, 1-200 chars) - Title of the todo
- `description`: Text (Optional, max 1000 chars) - Detailed description
- `completed`: Boolean (Default: False) - Completion status
- `created_at`: DateTime - Timestamp of creation
- `updated_at`: DateTime - Timestamp of last modification

## Relationships
- **User (1) <-> Todo (N)**: Each user can have multiple todos; each todo belongs to exactly one user.

## Validation Rules
- Title cannot be empty and must be <= 200 characters.
- Description must be <= 1000 characters.
- All requests to modify or delete a Todo must verify that the `user_id` matches the authenticated `subject`.
